定义：智能体是以大模型为 “大脑”，通过规划、记忆、工具调用、执行、反馈形成闭环，能自主感知、决策、行动、迭代的 AI 系统。
区别：纯大模型会话：被动响应、无状态、单次 / 有限上下文、仅生成文本、无执行能力。agent 主动规划、有状态（短期 / 长期记忆）、多轮循环、可调用工具 / API、能落地执行、可观测可审计。
核心能力：
    (1) 感知与理解：理解用户意图、环境状态、上下文。
    (2) 任务规划：拆解目标、制定步骤、动态调整（ReAct/Plan-and-Solve）。
    (3) 记忆管理：短期上下文、长期向量知识库、用户偏好。
    (4) 工具调用：API/MCP、数据库、代码执行、外部服务。
    (5) 自主执行：无需人工干预完成多步骤任务。
    (6) 反馈迭代：根据结果优化后续决策。
具体来讲就是给大模型增加：
    （1）知识增强 / RAG（专业知识库、检索增强）
    （2）记忆管理（短期上下文、长期向量库）
    （3）工具 / 服务调用（MCP、API、函数调用）
    （4）规划与编排（任务拆解、状态流转、循环执行）
    （5）运行时与治理（部署、监控、安全、审计）

















AI历史数据存储中心：相当于LangChain或Dify生态中的Memory组件。
    作用：管理历史会话， 数据持久化，数据生命周期管理
    GZIP适合大数据尤其文本数据（JSON）的压缩， 压缩效率能达到70%左右： 核心时LZ77算法的重复发现和霍夫曼算法的统计编码优化
AI mcp服务： 相当于LangChain的tools组件
    作用： 遵循MCP规范， 屏蔽业务服务接口、协议、通信细节差异， 让AI识别和使用（请求）业务数据服务更加标准和简单；
